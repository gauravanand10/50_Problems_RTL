# Day 9 - Dataflow Modeling

## Objective

Learn the Dataflow Modeling style in Verilog by implementing combinational circuits using continuous assignments (`assign`). Understand how Boolean expressions are used to describe hardware without using procedural blocks such as `always`.

## Design

### 1-Bit Full Adder (Dataflow Modeling)

Inputs:

- Input A (`a`)
- Input B (`b`)
- Carry Input (`cin`)

Outputs:

- Sum (`sum`)
- Carry Output (`cout`)

### Binary to Gray Code Converter

Inputs:

- N-Bit Binary Input (`binary_in`)

Outputs:

- N-Bit Gray Code Output (`gray_out`)

Parameter:

- Width (`N`)

## Flow

1. Learn Dataflow Modeling
2. Understand Continuous Assignment (`assign`)
3. Implement a 1-Bit Full Adder
4. Write Boolean Expressions for Sum and Carry
5. Create Testbench
6. Simulate the Design
7. Verify Full Adder Truth Table
8. Implement a Binary to Gray Code Converter
9. Verify Binary to Gray Code Conversion

## Learnings

- Dataflow Modeling
- Continuous Assignment (`assign`)
- Boolean Expressions
- Combinational Logic
- XOR (`^`) Operator
- AND (`&`) Operator
- OR (`|`) Operator
- Parameterized Modules
- Binary to Gray Code Conversion

## Key Observation

During simulation:

- Outputs change immediately whenever the inputs change.
- No clock signal is required.
- No memory elements are used.
- The output is continuously driven by Boolean expressions.
- Dataflow Modeling is ideal for combinational circuits.

## Assignment

Design a Binary to Gray Code Converter in Verilog using Dataflow Modeling. The module should take an N-bit binary input (`binary_in`) and generate the corresponding N-bit Gray code output (`gray_out`) using continuous assignments and basic logic operators.

## Assignment Design

### Binary to Gray Code Converter

Inputs:

- N-Bit Binary Input (`binary_in`)

Outputs:

- N-Bit Gray Code Output (`gray_out`)

Parameter:

- Width (`N`)

## Assignment Flow

1. Create a Parameterized Verilog Module
2. Declare N-Bit Binary Input
3. Declare N-Bit Gray Code Output
4. Implement Gray Code Conversion Using XOR Operations
5. Create Testbench
6. Apply Different Binary Input Values
7. Simulate the Design
8. Observe Gray Code Output
9. Verify Correct Binary to Gray Code Conversion

## Assignment Learnings

- Parameterized RTL Design
- Continuous Assignment (`assign`)
- Binary to Gray Code Conversion
- XOR-Based Logic Design
- Combinational Circuit Design
- Dataflow Modeling Style
- Testbench Development
- Waveform Analysis

## Key Observation

For every binary input:

- The Most Significant Bit (MSB) of the Gray code remains the same as the binary input.
- Every remaining Gray code bit is generated by XORing two adjacent binary bits.
- Only one bit changes between two consecutive Gray code values.
- The converter produces the correct Gray code immediately without requiring a clock.

## Conclusion

Dataflow Modeling provides a simple and efficient method for designing combinational circuits using Boolean expressions and continuous assignments. By implementing a 1-Bit Full Adder and a parameterized Binary to Gray Code Converter, this exercise demonstrates how combinational logic can be described clearly without using procedural blocks while ensuring immediate output updates whenever the inputs change.
